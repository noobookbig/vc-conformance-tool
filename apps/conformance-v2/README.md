# conformance-v2

The **v2** conformance test engine for the Thailand VC OID4VCI / OID4VP
1.0 Final test suite. Sibling to `apps/web/` (the v0.1.0 webapp); lives
on its own branch and does not modify any v0.1.0 code.

This is the **engine** workstream of [MAS-242](/MAS/issues/MAS-242). The
**server** (HTTP + SSE), **web UI** (Vite + React), **Docker image**, and
**v2.0.0 release** are follow-up issues:

- [MAS-255](/MAS/issues/MAS-255) — v2 server (HTTP API + SSE wrapping this engine)
- [MAS-256](/MAS/issues/MAS-256) — v2 web UI (Vite + React SPA)
- [MAS-257](/MAS/issues/MAS-257) — v2 release (Docker + GHCR + CHANGELOG)
- [MAS-258](/MAS/issues/MAS-258) — v2 QA gate

## What this fixes

The v0.1.0 webapp (and v0.1.0 of this repo) had a real bug:

> The runner reported `passRate: 1` (100% pass) against a closed port,
> because shape-only coverage cases inflated the denominator. See
> [MAS-219](/MAS/issues/MAS-219) (BLOCKER) and [MAS-220](/MAS/issues/MAS-220)
> (fix) for the full root-cause.

The v2 engine is the **structural** fix:

1. **No shape-only "passes".** Every `live` case contacts a real target
   (or the in-process mock). `coverage` cases are a separate, lower
   weight, must carry a `justification` string naming the spec section.
2. **The catalogue loader rejects any suite where more than 50% of
   cases are `coverage`.** This is the structural guard that prevents
   the v0.1.0 inflation pattern from returning.
3. **Stop-on-error is mandatory by default.** A real failure latches the
   `AbortCoordinator` and stops the suite. The CLI exits 3. No more
   "demo UX" that collects failures forever.
4. **Precheck is a separate gate.** The runner calls `precheck()` before
   iterating the catalog. An unreachable target fails fast with exit 4
   (distinct from a per-case stop-on-error).
5. **Per-case rows carry the response status, body, and the failing
   assertion.** Reports (JSON / JUnit / HTML) make the failure
   self-documenting — no need to re-derive from logs.

## Exit codes

| Code | Meaning |
| ---: | --- |
| 0 | Full pass. Every case in the catalog passed; no skips, no aborts. |
| 2 | Skipped only. Every case either passed or was skipped; no failures, no aborts. |
| 3 | Halted. A real failure triggered stop-on-error; `abortedAt: <case-id>`. |
| 4 | Precheck failed. Target was unreachable; the catalog loop never ran. |

These four codes are the **stable contract** the Server workstream
(MAS-255) wraps as `POST /api/runs` and the Docker image (MAS-257)
exits with. Do not change them without a CTO sign-off.

## Run

### Regenerate the catalog

```bash
# From the repo root.
node scripts/parse-testcases.mjs \
  --in  /home/big/Documents/vc-test/docs/conformance/openid4vci-vp/conformance-testcase-corrected.md \
  --out references/testcases
```

The script is idempotent and re-emits all 317 cases (217 live, 100
coverage) under `references/testcases/`. The script also re-validates
the output against the loader's contract and exits non-zero if fewer
than 30 cases were written or if coverage exceeds 50%.

### Run the engine against the in-process mock

```bash
cat > /tmp/v2-mock.yaml <<'EOF'
useMock: true
EOF

rm -rf /tmp/v2-out
npm run v2:run -- --config /tmp/v2-mock.yaml \
  --catalog references/testcases --out /tmp/v2-out
echo "exit: $?"
# → exit 0
#   wrote /tmp/v2-out/report.json
#   wrote /tmp/v2-out/report.junit.xml
#   wrote /tmp/v2-out/report.html
```

### Run against a real target

```bash
cat > /tmp/v2-real.yaml <<'EOF'
targetIssuer: http://127.0.0.1:8080
useMock: false
EOF

npm run v2:run -- --config /tmp/v2-real.yaml \
  --catalog references/testcases --out /tmp/v2-out
```

When `targetIssuer` is set, the runner makes a `GET /case/<id>` for
every live test case and treats 2xx as pass. A 4xx / 5xx / timeout /
refused counts as a real failure and trips the abort.

### Run against a target that is down

```bash
cat > /tmp/v2-dead.yaml <<'EOF'
targetIssuer: http://127.0.0.1:1/never
useMock: false
EOF

npm run v2:run -- --config /tmp/v2-dead.yaml \
  --catalog references/testcases --out /tmp/v2-out
echo "exit: $?"
# → exit 4 (precheck failed)
```

The precheck report is written to `/tmp/v2-out/report.json` with
`abortedAt: "precheck"`, `error: "target unreachable"`, and an empty
`results: []`.

## Tests

```bash
npm run v2:test
```

7 spec files, 45 tests:

- `abort.test.ts` — latch, idempotency, exit-code contract.
- `loader.test.ts` — YAML parsing, validation, the >50% coverage guard.
- `http.test.ts` — fetch wrapper, timeout, 4xx/5xx body capture, refused.
- `precheck.test.ts` — closed port, in-process mock, 4xx/5xx.
- `stop-on-error.test.ts` — first failure halts the suite; `abortedAt`.
- `wi-mock.test.ts` — all live cases pass against an in-process mock.
- `report.test.ts` — JSON, JUnit XML, HTML writers; escape correctness.

## Event stream

The runner emits a typed event stream the Server workstream wraps as
SSE. The order is:

```
run.started
case.passed*  (one per passing case)
case.failed   (the one failing case that latched the abort)
case.skipped* (skipped cases; do not affect abort)
run.aborted
```

If the suite runs to completion without a real failure, `run.aborted`
is not emitted; `run.completed` is. The CLI prints these to stderr
(`2>/dev/null` quiets them in CI).

## Relationship to v0.1.0

The v0.1.0 webapp (`apps/web/`) ships as-is. It is in active use, has
an accepted MAS-220 patch, and a 120/120 green test suite. The v2
work **does not modify** any v0.1.0 code; it adds a sibling package.

The v0.1.0 webapp's runner (in `apps/web/src/runners/runner.ts`) is
the older "shape-only with coverage" model. The v2 engine is the
"real per-case pass/fail, halts on real error" model. They are two
different tools, both shipped, both maintained.

## Catalog contract (v2)

A test case is one YAML file. Required fields:

| Field | Type | Notes |
| --- | --- | --- |
| `id` | string | Stable id; usually the v2.0 spec id (e.g. `FT.IC.AU.I.H.VB.001`). |
| `name` | string | Human-readable name. |
| `operation` | string | Operation grouping (e.g. `Issue VC — Authorization`). |
| `eut` | enum | `issuer` / `verifier` / `wallet` / `holder` / `resolver` / `multi`. |
| `suite` | enum | `holder` / `issuer` / `verifier` / `multi`. |
| `behavior` | enum | `valid` / `invalid`. |
| `kind` | enum | `live` (default) / `coverage`. `coverage` requires `justification`. |
| `justification` | string? | Required when `kind: coverage`. Names the spec section. |
| `specRef` | string? | Free-form spec reference from the corrected spec. |
| `sourceFile` | string? | Filename of the source markdown (traceability). |

The structural guard `loadCatalog()` enforces:

- empty directory → reject
- >50% `coverage` → reject
- duplicate ids → reject
- missing required fields → reject
